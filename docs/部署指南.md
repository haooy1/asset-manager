# IT资产管理系统 - 部署指南

## 快速安装（推荐）

### 🐧 Linux 一键安装（麒麟/V10/欧拉/Ubuntu）

```bash
# 下载安装脚本
curl -o install.sh https://raw.githubusercontent.com/haooy1/asset-manager/main/scripts/install.sh

# 一键安装（自动安装 Node.js + PostgreSQL + 初始化 + 配置服务）
sudo bash install.sh
```

安装完成后访问 `http://localhost:3000/login`，默认管理员 `admin / admin123`

### 🪟 Windows 一键安装

下载安装包后双击 `install.bat`，脚本自动完成环境检测、依赖安装、数据库初始化、服务启动。

## 一、首次部署

### 1. 安装 Docker

```bash
# 银河麒麟 V10 / 欧拉 EulerOS
sudo yum install -y docker-ce docker-compose-plugin
sudo systemctl enable docker --now

# Ubuntu / 银河麒麟桌面版
sudo apt update && sudo apt install -y docker.io docker-compose-v2
sudo systemctl enable docker --now
```

### 2. 创建部署目录

```bash
sudo mkdir -p /opt/asset-manager/uploads
cd /opt/asset-manager

# 下载 docker-compose.prod.yml
curl -O https://raw.githubusercontent.com/haooy1/asset-manager/main/docker-compose.prod.yml
# 或从部署包中复制
```

### 3. 配置环境变量

```bash
# 创建 .env 文件
cat > .env << EOF
DATABASE_URL=postgresql://asset_user:ChangeMe123@db:5432/asset_manager?schema=public
DB_USER=asset_user
DB_PASSWORD=ChangeMe123
DB_NAME=asset_manager
AUTH_SECRET=$(openssl rand -base64 32)
AUTH_URL=http://your-server-ip:3000
EOF
```

### 4. 启动服务

```bash
docker compose -f docker-compose.prod.yml up -d
```

### 5. 初始化数据库

```bash
docker exec -it asset-manager npx prisma db push
```

### 6. 创建初始管理员

```bash
# 进入容器
docker exec -it asset-manager node -e "
const { PrismaClient } = require('@prisma/client');
const bcrypt = require('bcryptjs');
const db = new PrismaClient();
(async () => {
  const hash = await bcrypt.hash('admin123', 12);
  await db.user.create({
    data: {
      username: 'admin',
      password: hash,
      realName: '系统管理员',
      role: 'SUPER_ADMIN',
    }
  });
  await db.branch.create({
    data: { name: '总部', code: 'HQ' }
  });
  console.log('管理员创建成功！用户名: admin, 密码: admin123');
})();
"
```

7. 访问 `http://your-server-ip:3000/login`

## 二、系统升级

### 方式一：一键升级脚本

```bash
# Linux / 麒麟 / 欧拉
cd /opt/asset-manager
bash scripts/upgrade.sh [版本号]

# Windows
cd C:\asset-manager
scripts\upgrade.bat [版本号]
```

升级脚本会自动：备份数据库 → 拉取新镜像 → 重启服务 → 执行数据库迁移

### 方式二：手动升级

```bash
# 1. 备份数据库
docker exec asset-manager-db pg_dump -U asset_user asset_manager > backup_$(date +%Y%m%d).sql

# 2. 拉取新版本
docker pull haooy1/asset-manager:latest

# 3. 重启服务
docker compose -f docker-compose.prod.yml down
docker compose -f docker-compose.prod.yml up -d

# 4. 迁移数据库
docker exec -it asset-manager npx prisma db push
```

## 三、日常维护

### 查看日志

```bash
docker logs -f asset-manager
docker logs -f asset-manager-db
```

### 重启服务

```bash
docker compose -f docker-compose.prod.yml restart
```

### 数据备份

```bash
# 备份整个数据库
docker exec asset-manager-db pg_dump -U asset_user asset_manager > /backup/asset_$(date +%Y%m%d).sql

# 恢复
cat /backup/asset_20260507.sql | docker exec -i asset-manager-db psql -U asset_user asset_manager
```

### 磁盘清理

```bash
# 清理旧镜像和构建缓存
docker system prune -f

# 定期清理上传目录中的旧文件（根据需要）
find /opt/asset-manager/uploads -mtime +365 -delete
```

## 五、防火墙配置

```bash
# 内网部署，仅开放 3000 端口给内网用户
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="192.168.0.0/16" port port="3000" protocol="tcp" accept'
sudo firewall-cmd --reload

# 或简单开放（仅在纯内网环境）
sudo firewall-cmd --add-port=3000/tcp --permanent
sudo firewall-cmd --reload
```

## 六、Nginx 反向代理（可选）

若需 SSL 或域名访问，可配合 Nginx：

```nginx
server {
    listen 80;
    server_name asset.yourcompany.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
