# 说明文档

## 一、项目概述

**项目名称**：IT 硬件资产全生命周期管理系统 (asset-manager)

**核心价值**：为中型企业提供 IT 硬件资产的统一管理平台，覆盖设备入库、领用、维保、报废全流程，支持多分支机构独立管控与安全合规文档管理。适配国产操作系统（银河麒麟/欧拉）。

**技术栈**：Next.js 16 + TypeScript + PostgreSQL + Prisma + NextAuth.js + Tailwind CSS + Docker

---

## 二、项目规划

### 2.1 功能范围

| 模块 | 功能 |
|------|------|
| 资产管理中心 | 8大品类（办公电脑/外设/网络/服务器/移动/会议/安全设备/安全文档）CRUD，状态流转 |
| 审批工作流 | 领用→归还→调拨→报废 完整闭环（申请→审批→执行） |
| 到期提醒 | 维保到期/证书到期自动扫描，node-cron 每日定时提醒 |
| 批量导入导出 | CSV 模板下载、批量导入（校验+错误报告）、数据导出 |
| 组织权限 | 总部/分支/部门三级架构，SUPER_ADMIN/BRANCH_ADMIN/DEPT_MANAGER/EMPLOYEE 四角色 |
| 移动端适配 | H5 响应式布局，底部导航栏，汉堡菜单 |
| 审计日志 | 关键操作自动记录（创建/更新/删除/审批） |
| 版本升级 | Docker 多架构镜像 + 一键升级脚本 |

### 2.2 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 全栈框架 | Next.js 16 (App Router) | 前后端 TypeScript 统一，国产系统兼容 |
| 数据库 | PostgreSQL 15+ | 成熟稳定，麒麟/欧拉原生支持，RLS 数据隔离 |
| ORM | Prisma | 类型安全，自动迁移，与 Next.js 深度集成 |
| 认证 | NextAuth.js (Credentials+JWT) | 轻量，支持多角色，无需第三方 |
| UI | Tailwind CSS | 响应式便捷，移动端友好 |
| 部署 | Docker + docker-compose | 一份镜像适配 x86_64 + ARM64 |

## 四、部署说明

### 4.1 环境要求
- Node.js >= 20.9
- PostgreSQL 15+
- Docker 20.10+（生产部署）

### 4.2 开发启动
```bash
pnpm install
cp .env.example .env.local     # 配置 DATABASE_URL 和 AUTH_SECRET
pnpm db:push                    # 初始化数据库
pnpm dev                        # 启动 → http://localhost:3000
```

### 4.3 生产部署
```bash
docker compose -f docker-compose.prod.yml up -d
docker exec -it asset-manager npx prisma db push
# 创建管理员（参考 docs/部署指南.md）
```

### 4.4 系统升级
```bash
bash scripts/upgrade.sh [版本号]
```

## 六、问题修复记录

### 2026-05-14：EMPLOYEE 角色资产权限隔离修复

**问题描述**：普通用户（EMPLOYEE 角色）在资产列表页能看到所有资产，而非仅属于自己的资产。资产详情页也存在越权访问风险。

**修复范围**：后端 API（路由 + 服务层）+ 前端 UI（按钮权限控制）

**修改文件清单**：

| 文件 | 修改内容 |
|------|----------|
| `src/app/api/assets/route.ts` | POST 改用 `requireRole("SUPER_ADMIN", "BRANCH_ADMIN", "DEPT_MANAGER")` 限制创建权限 |
| `src/app/api/assets/[id]/route.ts` | PUT/DELETE/POST 添加角色限制；GET 增加 EMPLOYEE 只能查看自己资产的校验 |
| `src/modules/assets/services.ts` | `getAssetList` 中 EMPLOYEE 自动过滤为 `assignedUserId = 当前用户ID` |
| `src/app/(dashboard)/assets/page.tsx` | 前端隐藏 EMPLOYEE 的 "+ 新增资产" 按钮 |
| `src/app/(dashboard)/assets/[id]/page.tsx` | 前端隐藏 EMPLOYEE 的 "编辑信息" 按钮 |

**修复的关键 Bug**：4 处 `requireRole(["A","B","C"])` 调用误传入数组，函数签名是 spread 语法 `requireRole(...roles)`，导致权限检查永为 false。已全部修正为 `requireRole("A","B","C")`。

**验证结果**（6/6 通过）：

| 测试项 | 预期 | 结果 |
|--------|------|------|
| EMPLOYEE 资产列表 | 仅见分配给自己的资产 | ✅ |
| EMPLOYEE 查看自己的资产详情 | 200 | ✅ |
| EMPLOYEE 查看他人资产详情 | 403 | ✅ |
| EMPLOYEE 创建资产 | 403 | ✅ |
| Admin 创建资产 | 201 | ✅ |
| EMPLOYEE 编辑资产 | 403 | ✅ |

**部署环境**：Kylin 服务器 `100.87.31.92:3000`（/opt/asset-manager），服务名 `asset-manager`

### 2026-05-15：安全文档自定义字段增强

**功能描述**：针对安全文档（SECURITY_DOCUMENT）类型，预置推荐自定义字段并优化前端展示。安全文档包含渗透测试报告、等保测评报告、风险评估报告等，需要体现报告时效性。

**新增默认字段**（6 个）：

| 字段标识 | 显示名称 | 类型 | 必填 | 说明 |
|----------|----------|------|------|------|
| `report_type` | 报告类型 | SELECT | ✅ | 渗透测试报告/等保测评报告/风险评估报告/... |
| `assessment_date` | 测评时间 | DATE | ✅ | 评测执行日期 |
| `assessment_org` | 测评机构 | TEXT | | 执行评测的机构名称 |
| `compliance_standard` | 合规标准 | SELECT | | GB/T 22239-2019/ISO 27001/等保2.0/... |
| `assessment_result` | 测评结论 | SELECT | | 符合/基本符合/不符合/不适用 |
| `certificate_no` | 证书编号 | TEXT | | 等保证书或其他合规证书编号 |

**前端优化**：
- 安全文档新增页：新增自定义字段渲染区域，自动加载 6 个推荐字段
- 安全文档详情页：隐藏无关字段（型号/品牌/存放位置/设备价值），日期标签改为"生效日期"/"到期日期"

**修改文件**：
| 文件 | 修改内容 |

| `src/app/(dashboard)/assets/new/page.tsx` | SECURITY_DOCUMENT 表单加入自定义字段渲染 |
| `src/app/(dashboard)/assets/[id]/page.tsx` | 详情页安全文档适配（隐藏硬件字段、日期标签重命名） |
| `scripts/seed_security_doc_fields.js` | 新增：安全文档推荐字段种子脚本 |

### 2026-05-15：自定义设备类型 + 自定义字段功能

**功能描述**：支持用户自定义设备类型（如 VR设备、打印机耗材），并为每种设备类型创建自定义字段。新增/编辑资产时动态渲染自定义字段并存储。

**新增数据模型**：

| 模型 | 说明 |
|------|------|
| `CategoryGroup` | 设备类型定义（内置 8 类 + 用户自定义） |
| `CustomField` | 自定义字段定义（关联 CategoryGroup） |
| `CustomFieldValue` | 资产自定义字段值（关联 Asset + CustomField） |
