# 产品概述

## 1. 项目愿景 (Vision)

> 为中型企业 IT 部门提供一套轻量、实用、可私有化部署的硬件资产全生命周期管理平台，告别 Excel 台账，让每台设备都有据可查、每个审批都有迹可循。

## 2. 核心价值 (Value Proposition)

*   **用户价值**：
    *   一屏掌控所有 IT 硬件资产，设备位置、归属人、状态实时可查
    *   领用/归还/调拨/报废全流程线上审批，告别纸质签单和口头沟通
    *   维保到期、证书到期自动提醒，不再被动应对
*   **商业/技术价值**：
    *   私有化部署，数据不出企业内网，满足安全合规要求
    *   多分支机构独立管控，数据天然隔离，权限灵活可控

## 3. 核心板块 (Core Modules)

### 3.1 资产管理中心
*   **描述**：IT 硬件资产的全生命周期管理，从入库到报废每个环节可追溯。
*   **关键特性**：
    *   支持八大资产品类：办公电脑、外设配件、网络设备、服务器存储、移动设备、会议设备、网络安全设备、安全文档
    *   设备全流程跟踪：入库 → 闲置 → 领用 → 维保 → 报废，状态自动流转
    *   每个设备记录：编号、名称、型号、品牌、购置日期、维保截止日、归属人、存放位置、当前状态
    *   安全文档类资产支持附件上传和有效期管理（如等保证书到期提醒）

### 3.2 审批工作流
*   **描述**：覆盖领用、归还、调拨、报废四大场景的线上审批。
*   **关键特性**：
    *   员工发起申请 → 部门负责人审批 → IT 管理员确认执行，逐级流转
    *   审批记录可回溯，支持审批状态（待审/通过/驳回）实时查看
    *   移动端 H5 支持快速审批，不在工位也能处理

### 3.3 到期提醒
*   **描述**：维保到期和证书到期的自动化提醒。
*   **关键特性**：
    *   设备维保截止日临近自动标记（如提前 30 天提醒）
    *   安全文档有效期到期提醒
    *   首页仪表板展示即将到期的设备和文档清单

### 3.4 组织与权限
*   **描述**：总部-分支-部门三级组织架构，多角色权限控制。
*   **关键特性**：
    *   各分支数据独立管理、互不可见
    *   支持管理员、部门负责人、普通员工三种基础角色（可根据需要扩展）
    *   分支管理员只能管理本分支的资产和审批

## 4. 目标用户 & 核心场景 (User & Scenes)

### 4.1 目标用户
*   **IT 管理员（总部/分支）**：负责资产录入、调配、维保跟踪、审批执行。核心诉求是"查得快、流程顺、不漏事"。
*   **普通员工**：发起领用/归还申请，查看名下设备。核心诉求是"操作简单、申请快"。
*   **部门负责人**：审批本部门员工的领用/归还请求。核心诉求是"审批方便（支持移动端）、信息清晰"。
*   **安全合规人员**：上传和管理安全文档，跟踪证书有效期。核心诉求是"集中归档、到期不遗漏"。

### 4.2 核心场景 (Key Use Cases)

*   **场景 1**：新员工入职，填写领用申请单 → 部门负责人审批 → IT 管理员分配设备并确认 → 设备状态自动变为"使用中"，归属到该员工名下。
*   **场景 2**：员工离职，发起归还申请 → IT 管理员确认设备归还 → 设备状态变为"闲置"，等待下一次分配。
*   **场景 3**：某台服务器维保即将到期（提前 30 天），首页仪表板自动提醒 → IT 管理员联系厂商续保或发起报废。
*   **场景 4**：杭州分支新购一批笔记本电脑，分支管理员批量录入系统 → 设备入库后进入"闲置"状态，可被申领。
*   **场景 5**：安全合规人员上传年度渗透测试报告，设置下一年度到期提醒 → 系统到期前自动提醒。

## 5. 关键业务流程 (Core Workflows)

### 设备生命周期主流程
```
设备入库（闲置）→ 员工领用申请 → 部门负责人审批 → IT管理员分配确认 
→ 设备状态变更为"使用中" → 维保到期提醒 → 员工归还申请 → IT管理员确认 
→ 设备状态变更为"闲置" → [循环分配] 或 发起报废审批 → 设备状态变更为"已报废"
```

### 审批流程
```
申请人提交申请 → 部门负责人审批（通过/驳回）→ IT管理员确认执行 → 完成
```
驳回时需填写驳回理由，通知申请人。全程记录操作人、操作时间、操作内容。

## 6. 验收标准 (Acceptance Criteria)

*   **功能完备性**：核心流程跑通（入库、领用、归还、报废）+ 基础审批（申请-审批-执行）+ 资产列表查询与筛选，无严重 Bug。
*   **技术指标**：PC Web 端页面加载 < 2 秒，移动端 H5 核心操作页面适配正常（扫码、审批）。
*   **数据规模**：支持 2000 台设备稳定运行，资产列表查询响应正常。
*   **权限隔离**：不同分支用户登录后只能看到本分支的资产数据。

## 7. 约束条件 (Constraints)

*   部署方式：私有化部署（公司内部服务器），外网不可访问
*   平台形态：PC Web + 移动端 H5 适配（非独立 App）
*   管理规模：500-2000 台设备
*   不接入第三方系统（不接钉钉、企业微信、飞书审批）

## 8. 项目边界 (Out of Scope)

*   暂不做财务级折旧计算（仅记录设备价值，不自动算折旧）
*   暂不做采购管理（不管供应商、采购合同、发票）
*   暂不做第三方系统对接（钉钉/企业微信/飞书等审批集成）
*   暂不做数据大屏 / BI 可视化看板
*   暂不支持多语言
