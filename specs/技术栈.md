# 技术栈选型

## 1. 核心决策 (Key Decisions)

本系统面向中型企业内部 IT 资产管理，核心约束为**私有化部署、国产操作系统适配（银河麒麟/欧拉）、PC Web + 移动端 H5 双端**。

选择 **Next.js 全栈 + PostgreSQL** 的理由：

* **前后端一体化**：Next.js 的 API Routes / Server Actions 可将前后端写在同一项目中，单一语言（TypeScript）贯穿全栈，AI 辅助开发效率最高，一个上下文即可完成完整功能的代码生成
* **国产系统友好**：Node.js 20.x 在银河麒麟 V10（APT 源、nvm、二进制包均可安装）和欧拉（RPM 源）上完全兼容，PostgreSQL 在麒麟桌面版有原生 APT 源、服务器版和欧拉上有 YUM 源，ARM64（飞腾/鲲鹏）和 x86_64 双架构均支持
* **成熟稳定**：所有选型均为业界广泛验证的生产级方案，文档丰富、社区活跃，遇到问题易于排查
* **Docker 容器化**：一份镜像通过 buildx 构建多架构版本，在麒麟/欧拉上均可一键部署，不依赖宿主系统软件源

## 2. 前端 (Frontend)

*   **框架**: Next.js 14（App Router）—— 文件系统路由，React Server Components，内置 Image/Font 优化
*   **语言**: TypeScript —— 类型安全，减少运行时错误，AI 生成代码质量更高
*   **样式方案**: Tailwind CSS —— 原子化 CSS，移动端响应式适配方便，AI 编写效率高
*   **组件库**: shadcn/ui —— 提供管理后台常用组件（表格、表单、弹窗、侧边栏、日期选择器），完全可定制，无障碍支持好
*   **图标**: lucide-react —— 与 shadcn/ui 配套，轻量矢量图标
*   **表单校验**: React Hook Form + Zod —— 声明式表单校验，TypeScript 类型推导

## 3. 后端 (Backend)

*   **运行时/语言**: Node.js 20.x LTS + TypeScript —— 前后端统一语言，国产化兼容性已验证
*   **Web 框架**: Next.js API Routes / Server Actions —— 与前端同框架，无需单独启动后端进程
*   **API 协议**: RESTful —— 资产管理系统 CRUD 场景为主，REST 最为直观，也便于 H5 端调用
*   **认证**: NextAuth.js (Auth.js v5) —— 内置 Credentials 登录 + JWT 会话，支持多角色权限控制
*   **定时任务**: node-cron —— 轻量级定时器，用于维保到期扫描和提醒推送
*   **文件上传**: 本地文件系统存储（Multer 适配）—— 内网私有化部署场景，附件直接存服务器磁盘

## 4. 数据存储 (Database)

*   **主数据库**: PostgreSQL 15+ —— 最成熟的开源关系型数据库，支持行级安全（RLS，可实现分支数据天然隔离），事务完整，生态丰富
*   **ORM**: Prisma —— TypeScript 原生 ORM，自动生成类型，Schema 可视化，迁移管理清晰，与 Next.js 集成度最高
*   **缓存 (可选)**: 暂不引入，500-2000 台设备量级下直接查询 PostgreSQL 即可满足性能要求

## 5. 基础设施 & 工具 (Infra & Tools)

*   **部署方式**: Docker 容器化部署 —— `docker buildx` 构建多架构镜像（linux/amd64, linux/arm64），在银河麒麟/欧拉上均可运行
*   **反向代理**: Nginx —— 静态资源缓存、HTTPS 终端、请求转发
*   **进程守护**: systemd（麒麟/欧拉原生）或 PM2 —— 服务崩溃自动重启、开机自启
*   **包管理**: pnpm —— 磁盘空间友好、安装速度快、严格的依赖管理
*   **代码规范**: ESLint + Prettier —— TypeScript 代码质量和格式统一
*   **版本控制**: Git
*   **npm 镜像**: npmmirror.com —— 国产网络环境下加速依赖安装（内网环境可配置私有镜像站）
