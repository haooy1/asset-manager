## 2. 关键文件说明 (Key Files)

*   **package.json**: 项目依赖管理文件，包含 Next.js、Prisma、NextAuth.js 等核心依赖。
*   **.env.example**: 环境变量示例，包含数据库连接串、JWT 密钥等（禁止提交真实值）。
*   **prisma/schema.prisma**: 数据库表结构定义，所有数据模型集中管理，通过 `prisma migrate dev` 生成迁移。
*   **next.config.ts**: Next.js 配置文件，包含图片域名、重定向、环境变量等。
*   **Dockerfile**: 多阶段构建，产出可直接在麒麟/欧拉上运行的生产镜像。
*   **docker-compose.yml**: 本地开发用，一键启动 PostgreSQL 容器。
*   **README.md**: 项目入口文档，必须包含"如何启动开发环境"、"如何部署"说明。

## 3. 模块说明 (Module Description)

*   **src/modules/assets/**: 资产管理核心逻辑。包含设备的增删改查、状态流转、品类枚举、导入导出等。
*   **src/modules/approvals/**: 审批流引擎。领用申请、归还确认、调拨、报废审批的状态机管理。
*   **src/modules/reminders/**: 到期提醒。维保到期、证书到期的扫描与推送逻辑。
*   **src/modules/org/**: 组织与权限。分支、部门、用户、角色的 CRUD 及数据隔离控制。
*   **src/shared/**: 通用工具与组件，不包含任何业务逻辑。shadcn/ui 组件通过 `npx shadcn-ui@latest add` 自动生成到 `shared/components/ui/`。
*   **src/lib/**: 基础设施层。数据库客户端、认证配置、定时任务等横切关注点。

## 4. 文件放置规则 (Placement Rules)

*   **页面/路由**：放在 `src/app/` 下，遵循 Next.js App Router 文件约定（`page.tsx`、`layout.tsx`、`loading.tsx`）。
*   **API 接口**：放在 `src/app/api/` 下，使用 `route.ts` 文件约定。
*   **业务逻辑**：放在 `src/modules/[模块名]/services.ts`，禁止直接在 UI 组件中写数据库查询或业务判断。
*   **数据库操作**：复用频率高的查询封装到 `src/lib/db/queries/`；一次性的简单查询可以直接在 `services.ts` 中写。
*   **通用组件**：放在 `src/shared/components/`，不依赖任何业务模块。
*   **类型定义**：业务相关类型定义就近放在 `src/modules/[模块名]/types.ts`；全局类型放在 `src/shared/`。
*   **附件上传**：安全文档等文件上传至 `public/uploads/`，数据库仅存储文件路径。
